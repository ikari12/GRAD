# Research Reproduction Pipeline

Fully reproducible analysis pipeline for the study:
**"Deconstructing Decoupling: Why Cardiac Drift Index Measures Route Geometry, Not Cardiovascular Fitness"**

## Quick Start

```bash
cd research
bash run_all.sh
```

This single command runs the entire pipeline: download → preprocess → analyze → verify.

## Directory Structure

```
research/
├── data/                        # Data (input + preprocessed)
│   ├── endomondoHR.json         # Raw FitRec dataset (6GB, downloaded by run_all.sh)
│   ├── meixner_4d_indices.csv   # Meixner 4D indices (generated by 00a)
│   ├── abc_metrics.csv          # ABC novel metrics (generated by 00b)
│   └── expected_values.json     # 31 expected values for verification
├── results/                     # Analysis output (generated by run_all.sh)
│   ├── study1.txt
│   ├── study2.txt
│   └── study3.txt
├── 00a_compute_4d.py            # Preprocessing: Meixner 4D indices from raw JSON
├── 00b_compute_abc.py           # Preprocessing: GACD, GradSens, SpeedSens from raw JSON
├── 01_study1_construct.py       # Study 1: Construct validity (ICC, PCA, split-half)
├── 02_study2_artifact.py        # Study 2: DI artifact detection (simulation, route R²)
├── 03_study3_variance.py        # Study 3: Variance decomposition (Person/Route/Occasion)
├── 04_verify.py                 # Automated verification against expected values
├── run_all.sh                   # Entry point (full pipeline)
└── requirements.txt             # Python dependencies
```

## Prerequisites

- Python 3.9+
- [Kaggle CLI](https://github.com/Kaggle/kaggle-api) (for data download only)

```bash
pip install -r requirements.txt
```

## Data Source

**FitRec dataset** by Ni et al. (2019)
- Kaggle: [tientd95/fitrec-dataset](https://www.kaggle.com/datasets/tientd95/fitrec-dataset)
- 253,020 workouts from 1,104 users with GPS, heart rate, altitude, and speed streams
- ~6GB uncompressed (`endomondoHR.json`)

The download is handled automatically by `run_all.sh`. If Kaggle CLI is not configured, manually download and place `endomondoHR.json` in `research/data/`.

## Pipeline Steps

| Step | Script | Input | Output | Time |
|------|--------|-------|--------|------|
| 0 | run_all.sh | — | Dependency check | <1s |
| 1 | run_all.sh | Kaggle API | `data/endomondoHR.json` | ~5 min |

## Flags

```bash
bash run_all.sh                           # Full pipeline
bash run_all.sh --skip-download           # Skip Kaggle download (data already present)
bash run_all.sh --skip-download --skip-preprocess  # Analysis only (CSVs already present)
```

## Cross-Validation

All cross-validation uses `GroupKFold(groups=userId)` to prevent data leakage from the same user appearing in both train and test folds. This is mandatory for repeated-measures data.

## Verification

`04_verify.py` checks 31 key values against `data/expected_values.json` with per-value tolerances. A successful run prints:

```
Summary: 31/31 values matched.
```

## Key Findings

| Metric | ICC | Occasion % | Interpretation |
|--------|-----|------------|----------------|
| GACD | 0.219 | 78.1% | Day-to-day condition dominates |
| GradSens | 0.358 | 58.5% | Moderate trait, strong occasion effect |
| SpeedSens | 0.431 | 56.9% | Moderate trait, strong occasion effect |
| DI (Meixner) | 0.163 | — | Poor reliability, route artifact |

DI is predicted from route geometry alone with R² = 0.82 (simulation) and R² = 0.60 (real hilly data), confirming it reflects gradient profile, not cardiovascular fitness.
